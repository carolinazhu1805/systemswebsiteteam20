Open-Illumiroom V2

For our research, our team conducted a detailed review of the previous years project: UCL Open-Illumiroom V2[1]. As our project builds upon this project, its main features are almost identical to ours. However, their version of Open-Illumiroom included hard-coded projections and triggers. By reviewing their work, we were able to better understand how to extend their project to add in the triggers and learn about the decisions behind previous implementations of Illumiroom.

Illumiroom by Microsoft research 2013

We then proceeded to read a paper published by Microsoft in 2023 [2]. This allowed us to further develop a deeper familiarity with our project. The paper discusses the reason behind Illumiroom and provided an in depth review of the technical considerations behind Illumiroom. This paper discusses the potential trigger applications of Illumiroom, which we found very relevant to our project. However, it mentions the design of games specifically for Illumiroom and how triggers can be easily created when having access to the source code of these games. Due to time and resource limitations, our team is unable to do that as we do not possess access to video game source codes. The paper then delves into other possibilities to program triggers, which we will attempt to implement:

  • Real-time image analysis of the rendered game
  • Controller input

    • Technology review

    We will now discuss our technical choices made during our work process.

    Possible solutions

    For the first part of our project, which was the integration of the two different Illumiroom modes within a single User Interface, our team spent time deciding what would be the best way to implement this. We considered the following:

  • Addition of a checkbox to indicate the user wants to use the command centre mode instead
  • Have a new page with two buttons - one to open Command centre and another one to open Immersive Graphics mode

    • We then decided to implement the former. We sent out a survey asking people which option they think would be best and results indicated the inclusion of a checkbox would be better. This is because this implementation allows the user to work on settings for both modes simultaneously and allows user to open the Command centre mode more easily, in less steps. For the second part of our project, we had to consider which triggers we wanted to implement and how. Our first confirmed trigger was the colour analysis, as we could use the suggestion from the previously read research paper of implementing a real time image analysis of the rendered game. We then considered weather to use the controller input, but decided to implement other triggers as we believe that it is harder to adapt controller input into separate games and controller output cannot be utilised for movies. As we want our project to be extendable into multiple movies and games, we brainstormed other triggers which would be more appropriate towards our goal. We decided to implement the following triggers: Colour, audio, time and later on due to additional time, implemented text recognition, further discussions in the choice of algorithms for each trigger will be made in the following section.
